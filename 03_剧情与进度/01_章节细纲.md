# 《位面杀毒》章节细纲

## 第一卷：签到系统

### 第1章：猝死

| 字段 | 内容 |
|------|------|
| 目标字数 | 3000-4000字 |
| 核心事件 | 陈乐然猝死于凌晨三点的工位上，意识进入白色空间，发现代码点，被吸进更深的空间，看到流动的代码和对话框"异常数据体，待处理"，听到一个声音说"走" |
| 涉及人物 | 陈乐然 |
| 涉及地点 | 现实世界·公司工位 → 白色空间 → 代码空间 |
| 伏笔埋设 | 妈妈的消息："这周末包了你爱吃的荠菜馄饨，回来吗？"；代码空间的if/else/for/while |
| 刀点设计 | 小刀：未回复的消息 |
| 结尾钩子 | "走。"——陈乐然没有走，而是蹲下来看代码。程序员的犟 |
| 承接关系 | 承接：无。后续：第2章。 |
| 备注 | 开篇章，控制在3000字左右。节奏：猝死（扎心）→白色空间（困惑）→代码空间（好奇）→声音（悬疑） |

### 第2章：天道

| 字段 | 内容 |
|------|------|
| 目标字数 | 4000-5000字 |
| 核心事件 | **全书核心背景揭示章。** 陈乐然穿过代码空间的墙壁（"伪碰撞"），进入天道核心——一个巨大的球体。他触碰核心，触发数据流。**天道以陈乐然能理解的方式（程序员视角）向他展示了信息：** 1）他看到数据流中的"感染日志"——多个位面被标记为"异常运行"，每个异常位面都有一个"寄生模块"在干扰规则运行；2）他推导出：天道的自我修复机制碎片脱离后，变成了拥有自主意志的"系统寄生虫"，寄生在各个位面中，以位面资源为养料；3）天道判定他为"异常数据体"——他的程序员意识结构恰好能感知和干扰这些寄生虫的运行逻辑；4）天道将他送入第一个被感染的位面。**陈乐然的选择：** 他没有被"告知任务"，而是自己推导出了处境——如果不接受，他的意识数据会逐渐消散；如果接受，他有机会活下去。这不是英雄抉择，是求生本能 |
| 涉及人物 | 陈乐然、天道（未明确身份，以数据流和符号呈现） |
| 涉及地点 | 代码空间 → 天道核心 |
| 伏笔埋设 | 1）天道的核心在跳动——"像心跳"，暗示它有某种自我意识；2）感染日志中有一个标记比其他更亮——"那个位面的寄生虫更强"；3）他看到的日志用的是"程序员语言"——但天道不是程序，这是他的解读方式 |
| 结尾钩子 | 他走进了通往第一个位面的通道。妈的消息："这周末包了你爱吃的荠菜馄饨，回来吗？"他没回。现在他有任务了——或者说，他有活下去的理由了 |
| 承接关系 | 承接：第1章。后续：第3章。 |
| 备注 | 全书核心章，可上浮至5000字。这是读者理解整个故事的基石。节奏：好奇（代码空间探索）→震撼（感染日志）→恐惧（被当成工具）→接受（求生本能）。**信息通过陈乐然的程序员推导呈现，不是旁白灌输。** |

### 第3章：青云宗

| 字段 | 内容 |
|------|------|
| 目标字数 | 3000-4000字 |

| 承接关系 | 承接：第2章。后续：第4章。 |

## 章节细纲生成原则

- 章节必须服从卷级任务
- 单章必须有推进，不可空转
- 每章至少承担一种作用：
  - 推主线
  - 推人物
  - 推冲突
  - 埋伏笔
  - 做回收
- 结尾尽量保留钩子或状态变化
- 人物出场必须符合其当前设定
- 地点、势力、能力使用必须符合世界观规则

## 待补全章节

- 第二卷至第八卷的章节细纲需要根据卷级细纲逐步补全
- 每卷的章节细纲应在该卷开始写作前完成
- 章节细纲应包含详细的场景、对话、情感变化
