# 《位面杀毒》卷级细纲

## 第一卷：签到系统（第1-50章）

| 维度 | 内容 |
|------|------|
| 字数 | 约20万字 |
| 核心任务 | 入局学习，掌握规则感知能力，完成第一个杀毒任务 |
| 碎片获得 | 签到碎片（签到系统被消灭后释放） |
| 能力觉醒 | 规则感知→规则编译·初阶（将已掌握规则跨位面翻译为战斗力） |
| 爽点设计 | 看穿爽：规则感知看到签到系统的灵气抽取流向；反杀爽：利用系统BUG断开林寂连接；战斗爽：破绽洞察指出弟子暗伤、灵气反噬让林寂自食其果；成长爽：签到碎片融入，能力觉醒 |
| 情感基调 | 荒诞喜剧（70%搞笑，30%扎心） |
| 结尾状态 | 小石头修炼速度恢复正常，获得签到碎片，能力觉醒为规则编译·初阶，碎片共鸣：林寂断开连接时的茫然 |
| 刀点设计 | 类型：小刀；位置：第45章；目标：小石头的无辜受害 |

### 卷内分段

| 段落 | 章节 | 任务 |
|------|------|------|
| 第一段 | 第1-3章 | 陈乐然猝死，进入天道核心，天道以程序员视角向他展示位面感染日志——多个位面被"系统寄生虫"感染。陈乐然被判定为异常数据体，天道将他送入第一个被感染的位面。他接受任务（没有别的选择），带着"找到并消灭系统寄生虫"的明确目标进入青云宗 |
| 第二段 | 第4-8章 | 在青云宗适应身份，观察签到系统运行机制，在茅厕墙上写下第一段伪代码，计算系统"工资率"，认识小石头，发现系统不仅剥削宿主还伤害无辜 |
| 第三段 | 第9-20章 | 深入分析签到系统的灵气抽取机制，发现林寂利用系统力量压制同门、收买人心、孤立小石头，陈乐然记录系统如何放大人性弱点 |
| 第四段 | 第21-35章 | 陈乐然制定规则编译方案，与林寂沟通（发现他主动选择依赖系统），道德困境——他有权替林寂做决定吗？最终决定继续，因为系统在伤害小石头 |
| 第五段 | 第36-45章 | 执行规则编译方案，引导林寂自愿断开连接，灵气反噬战斗场景，小石头修炼恢复 |
| 第六段 | 第46-50章 | 任务完成，离开位面，获得签到碎片，能力觉醒为规则编译·初阶，碎片共鸣 |

### 核心场景

**灵气反噬场景（战斗型爽感）：** 林寂发现陈乐然在暗中破坏签到系统，用签到系统积蓄的灵气全力攻击。灵气化作一道光柱轰向陈乐然。陈乐然没有躲，只是伸出一根手指，灵气在他指尖三寸处自动消散。林寂瞪大眼睛："怎么可能？"陈乐然在伪代码本上写下：`// 签到系统灵气抽取函数有个BUG——当目标距离小于三寸时，灵气自动回流到抽取源。`林寂不信邪，再次攻击。灵气再次回流，打在他自己身上，他吐血倒地。陈乐然看着他：`// 我没打你。你打的自己。`

**小石头恢复场景（情感爽感）：** 签到系统被断开后，小石头的修炼速度恢复正常。他第一次感受到灵气涌入身体的充实感，而不是每天被抽取的空虚感。他看着自己的双手，眼中亮起光。陈乐然在远处看着，伪代码：`// 一个孩子的修炼速度恢复正常了。这件事值得写下来。`

## 第二卷：抽奖系统（第51-100章）

| 维度 | 内容 |
|------|------|
| 字数 | 约20万字 |
| 宿主类型 | 系统失控型（周远） |
| 核心任务 | 深化理解，掌握规则编译·初阶，面对第一个"被系统催化异化"的宿主 |
| 碎片获得 | 抽奖碎片（抽奖系统被消灭后释放） |
| 能力觉醒 | 规则编译·初阶→规则编译·中阶（同时编译多个位面规则） |
| 爽点设计 | 看穿爽：用规则编译看到抽奖系统的底层逻辑（恐惧值、概率、隐藏规则）；战斗爽：让丧尸互相撕咬，不动手清场；反杀爽：制造系统死机让周远直面自己；成长爽：抽奖碎片融入，能力觉醒 |
| 情感基调 | 紧张反思（50%紧张，30%反思，20%苦涩） |
| 结尾状态 | 周远被迫直视自己的异化，获得抽奖碎片，能力觉醒为规则编译·中阶，碎片共鸣：周远坐在地上看着沾血的手 |
| 刀点设计 | 类型：中刀；位置：第90章；目标：周远不记得被放弃者的名字 |
| 核心冲突 | 周远不是被骗的，他是被系统催化成"最有效率版本"的好人——而最有效率的版本往往是最冷酷的 |

### 卷内分段

| 段落 | 章节 | 任务 |
|------|------|------|
| 第一段 | 第51-60章 | 进入末世位面，发现周远是"救世主"，抽奖系统与末世环境的共生关系 |
| 第二段 | 第61-70章 | 规则编译分析抽奖系统，发现恐惧驱动机制；观察到周远的决策风格开始变化 |
| 第三段 | 第71-80章 | 见证周远的异化：主动放弃外围幸存者去抽十连抽，计算后放弃受伤幸存者 |
| 第四段 | 第81-90章 | 制造"系统死机"，周远被迫直视自己——他不记得被放弃者的名字了 |
| 第五段 | 第91-100章 | 周远质疑系统也质疑自己，拆终端做收音机；陈乐然离开，获得抽奖碎片，碎片共鸣：周远坐在地上看着沾血的手 |

### 核心场景

**十连抽场景：** 外围幸存者通过无线电求救，丧尸正在逼近。周远犹豫了一秒——系统弹出"十连抽必出SSR"。他看了看求救信号，看了看抽奖界面，选择了抽奖。第二天他拿到了SSR武器。外围那批人没有回来。

**不记得名字场景：** 系统死机后，周远坐在地上看着沾血的手。他想起了那个被他放弃的受伤幸存者——那个人叫什么名字来着？他想了很久。他不记得了。系统没有删除他的记忆。他自己删除的。因为记住那个人的名字，不是最优解。

**丧尸失控场景（战斗型爽感）：** 丧尸围攻，周远要抽奖求生。系统弹出"十连抽必出SSR"。陈乐然一把按住周远的手："别抽。"周远："不抽我们都要死！"陈乐然看着丧尸群，在伪代码本上写下几行字，然后走向丧尸群。丧尸在他面前停下。然后，它们开始互相撕咬。周远目瞪口呆。陈乐然的伪代码：`// 丧尸目标锁定算法BUG——目标存在感降为0时，系统默认标记"同类"为新目标。`他转身看着周远：`// 所以我说了，别抽。解决问题不一定要花钱。`

## 第三卷：吞噬系统（第101-150章）

| 维度 | 内容 |
|------|------|
| 字数 | 约20万字 |
| 宿主类型 | 明知故犯型（冷酷少年） |
| 核心任务 | 认识代价，尝试规则编译·中阶突破，面对第一个"不愿被拯救"的宿主 |
| 碎片获得 | 未获得（系统未被击败，冷酷少年做出微小改变但系统继续运行） |
| 能力觉醒 | 能力停滞（未获得碎片） |
| 爽点设计 | 看穿爽：触发系统"呕吐"看到被吞噬前的世界；战斗爽：规则反噬让吞噬系统全力一击反弹自身，冷酷少年跪地；反杀爽（失败）：陈乐然无法说服对手；成长爽（紧迫）：未获得碎片，意识强度危机 |
| 情感基调 | 燃与沉（40%燃，40%沉，20%苦涩的希望） |
| 结尾状态 | 冷酷少年明知代价选择继续，系统未被击败，陈乐然带着没有答案的问题离开 |
| 刀点设计 | 类型：大刀；位置：第140章；目标：冷酷少年的"我知道，但我不放弃" |
| 核心冲突 | 冷酷少年完全清楚系统的代价，但因为经历过极端弱小，主动选择继续——陈乐然无权替他做决定 |

### 卷内分段

| 段落 | 章节 | 任务 |
|------|------|------|
| 第一段 | 第101-110章 | 进入高武位面，发现冷酷少年的过去（亲人被妖兽杀死）与吞噬系统的"进化"机制 |
| 第二段 | 第111-120章 | 逆向分析吞噬系统，发现存在删除的真相；观察到冷酷少年的噩梦 |
| 第三段 | 第121-130章 | 触发系统"呕吐"，让冷酷少年看到被吞噬前的世界——但他**关掉了画面** |
| 第四段 | 第131-140章 | 信念对决：冷酷少年拒绝被拯救，陈乐然无法说服一个逻辑自洽的对手 |
| 第五段 | 第141-150章 | 冷酷少年做出微小改变（选择性吞噬），不完美共存；陈乐然离开 |

### 核心场景

**关掉画面场景：** 陈乐然触发系统"呕吐"，让冷酷少年看到被吞噬前的世界——灵狐闪过林间，发光蝴蝶浮现空中。冷酷少年看了三秒钟，然后关掉了画面。"不用给我看。我一直都知道这个世界曾经是什么样子。""但那又怎样？那些灵狐不会保护我，那些蝴蝶不会救我的命。"

**信念对决场景：** 陈乐然试图断开连接，冷酷少年挡在面前："你要我放弃力量，回到那个什么都做不了的自己？""你说我在删除物种。那你能给我一个不删除物种就能变强的方式吗？"陈乐然沉默了。他给不出答案。

**招式崩溃场景（战斗型爽感）：** 冷酷少年用吞噬系统的全力一击，一拳轰向陈乐然。陈乐然没有躲，只是伸出一根手指，轻轻点在冷酷少年的手腕上。冷酷少年的拳头停在半空。他感觉自己的力量在流失——不是被吞噬，而是被"卸载"了。陈乐然的伪代码：`// 吞噬系统内存管理BUG——吞噬目标存在强度低于阈值时，系统误判为"已吞噬"，自动释放所有储存能量。`冷酷少年跪倒在地，体内储存的所有吞噬能量瞬间释放。他看着陈乐然，眼中第一次出现恐惧。陈乐然：`// 我没做什么。你的系统自己崩了。`

## 第四卷：好感度系统（第151-215章）

| 维度 | 内容 |

| 情感基调 | 正剧（60%正剧，30%沉，10%希望） |
| 结尾状态 | 获得好感度碎片，规则编译·高阶觉醒，侍卫想起妻子，王国陷入短期混乱，碎片共鸣：侍卫眼中亮起又熄灭的光 |
| 刀点设计 | 类型：暗刀；位置：第200章；目标：侍卫的真实情感被覆盖 + 顾行之的预言 |
| 核心冲突 | 顾行之主动用好感度系统统治王国，创造了二十年和平——代价是所有人的自由意志。陈乐然要打破这个"和平"吗？ |

### 卷内分段

| 段落 | 章节 | 任务 |
|------|------|------|
| 第一段 | 第151-165章 | 进入魔法位面，发现王国的"完美秩序"；遇到侍卫（受害者），发现好感度系统的意志清除机制；深入了解王国社会结构，发现底层民众的"幸福"是被覆盖的 |
| 第二段 | 第166-180章 | 规则编译分析好感度系统，发现刷新间隔漏洞；追踪系统源头，发现真正的宿主——摄政大臣顾行之；了解顾行之的过去（目睹战争惨状后主动寻求系统），理解他的动机 |
| 第三段 | 第181-195章 | 与顾行之正面交锋：信念之战——"二十年零战争"vs"自由意志"；顾行之调动被系统控制的军队追杀陈乐然；军队反噬战斗场景；陈乐然利用规则编译在追杀中生存，同时寻找系统弱点 |
| 第四段 | 第196-205章 | 利用规则编译瓦解情感覆盖；在刷新间隔中帮侍卫恢复一瞬真实意识（眼中亮起光又灭了）；逐步瓦解更多人的情感覆盖，引发王国秩序松动 |
| 第五段 | 第206-215章 | 系统瓦解，获得病毒样本；侍卫想起妻子；王国陷入短期混乱（验证顾行之的预言）；陈乐然离开 |

### 核心场景

**信念对决场景：** 顾行之与陈乐然对峙。"你说我剥夺了他们的自由意志。但你知道自由意志带来的是什么吗？战争、暴动、背叛。二十年零死亡。你要把这些还给他们？"陈乐然看着远处忠诚站岗的侍卫，看着他眼中偶尔闪过又熄灭的光，说："如果和平的代价是不再是人，那还是和平吗？"

**侍卫觉醒场景：** 陈乐然在好感度系统的刷新间隔中帮侍卫恢复了一瞬真实意识。侍卫想起了妻子——眼中亮起光。然后系统重新覆盖了。他眨了眨眼，继续忠诚站岗。那道光灭了。陈乐然在伪代码本上写下：**// 那道光不应该灭。**

**预言验证场景：** 系统瓦解后，王国确实陷入了短期混乱——贵族争权、平民恐慌、邻国试探。顾行之站在废墟中，看着混乱，没有得意，只有一种疲惫的确认："你赢了。但你看到了吧？这就是自由意志的代价。"陈乐然没有回答。但他没有后悔。

**军队反噬场景（战斗型爽感）：** 顾行之用好感度系统控制的军队围攻陈乐然。数百人同时出手，魔法、剑气、箭雨如潮水般涌来。陈乐然站在原地，没有动。他在伪代码本上写下一段代码，然后轻轻合上本子。所有攻击在距离他三米处同时停下。然后，它们开始反转——魔法回轰向法师，剑气回斩向剑士，箭雨回射向弓手。顾行之瞪大眼睛："你做了什么？"陈乐然的伪代码：`// 情感覆盖BUG——覆盖强度超过阈值时，被覆盖者"目标锁定"反转。他们现在攻击的不是我，是你。`

## 第五卷：选择推演系统（第216-265章）

| 维度 | 内容 |
|------|------|
| 字数 | 约20万字 |
| 宿主类型 | 受害者型（被困住的清醒者——早就知道系统有问题但无力挣脱，如释重负地请求陈乐然帮忙关掉） |
| 核心任务 | 理解自由意志，深化规则编译·高阶，完成第五个杀毒任务 |
| 碎片获得 | 推演碎片（选择推演系统被消灭后释放） |
| 能力觉醒 | 规则编译·高阶深化（自创全新规则组合） |
| 爽点设计 | 看穿爽：用规则编译看到选项控制的底层逻辑（"拒绝"选项被删除的UI渲染）；战斗爽：规则嫁接让系统把敌人包围圈标记为"危险区"；反杀爽：注入"不选"选项，让宿主看到从来不存在的选择；成长爽：推演碎片融入，高阶深化 |
| 情感基调 | 沉重（50%沉，30%反思，20%苦涩） |
| 结尾状态 | 宿主意识到自由意志被偷走，获得推演碎片，能力觉醒为规则编译·高阶深化，碎片共鸣：宿主从未自己做过一个决定 |
| 刀点设计 | 类型：回刀；位置：第255章；目标：自由意志的丧失 |

### 卷内分段

| 段落 | 章节 | 任务 |
|------|------|------|
| 第一段 | 第216-225章 | 进入都市位面，发现选择推演系统的选项控制机制 |
| 第二段 | 第226-240章 | 规则编译分析选择推演系统，发现"拒绝"选项被删除 |
| 第三段 | 第241-250章 | 注入"不选"选项，让宿主看到从来不存在的选项 |
| 第四段 | 第251-260章 | 宿主意识到自由意志被偷走，开始质疑系统 |
| 第五段 | 第261-265章 | 任务完成，离开位面，获得推演碎片，碎片共鸣 |

### 核心场景

**选项灰化场景：** 宿主面临一个重要商业决策，系统弹出两个选项——"接受合作"和"拒绝合作"。但"拒绝合作"的描述是灰色的，无法点击。宿主从未注意到这一点，因为他从未想过要拒绝。陈乐然在旁边看着，伪代码记录："选项灰化——系统直接删除了'拒绝'的UI渲染，用户根本看不到这个选项存在。"

**注入场景：** 陈乐然注入了第三个选项——"不选"。宿主第一次看到了从来不存在的选项，那个灰色的"拒绝合作"按钮突然亮了起来。他愣住了，伸手去点，手指悬在空中——他发现自己不知道该怎么"拒绝"。

**预判反制场景（战斗型爽感）：** 选择推演系统的宿主被对手围攻，系统弹出"最优逃跑路线"。陈乐然用规则编译看到预判算法的BUG，注入一条假指令——让系统把"逃跑路线"标记为"危险区域"。宿主本能地避开了所有"危险区域"，结果正好走到了对手的包围圈外。对手目瞪口呆：他怎么知道我们在这里埋伏？陈乐然的伪代码：`// 推演系统预判逻辑BUG——当"危险标记"与"最优路线"冲突时，系统优先避开危险。我只是把最优路线标记成了危险而已。`
